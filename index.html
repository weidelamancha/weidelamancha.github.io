Udacity Data Science Engineer Nano Degree Capstone Project

Posted on 2020-04-22

Project Definition

Project Overview

Once every few days, Starbucks sends out an offer to users of the mobile app. An offer can be merely an advertisement for a drink or an actual offer such as a discount or BOGO (buy one get one free). Every offer has a validity period before the offer expires. Some users might not receive any offer during certain weeks.

The data sets provided in the project contains simulated data that mimics customer behavior on the Starbucks rewards mobile app:

portfolio.json

  • id (string) - offer id
  • offer_type (string) - type of offer ie BOGO, discount, informational
  • difficulty (int) - minimum required spend to complete an offer
  • reward (int) - reward given for completing an offer
  • duration (int) - time for offer to be open, in days
  • channels (list of strings)

profile.json

  • age (int) - age of the customer
  • became_member_on (int) - date when customer created an app account
  • gender (str) - gender of the customer (note some entries contain ‘O’ for other rather than M or F)
  • id (str) - customer id
  • income (float) - customer’s income

transcript.json

  • event (str) - record description (ie transaction, offer received, offer viewed, etc.)
  • person (str) - customer id
  • time (int) - time in hours since start of test. The data begins at time t=0
  • value - (dict of strings) - either an offer id or transaction amount depending on the record

Problem Statement

There are a lots of potentials for using the simulated data for predictions. This blog post only covers the prediction of whether an offer is required for a customer with given demographics to purchase.
The prediction is made with a simple neural network using PyTorch. Then the model is deployed using AWS lambda and API gateway, so when a new customer login to the mobile app, a decision can be made whether an offer is needed to push to this customer depending on the member profile.

Ensemble Techniques

Posted on 2018-02-06

Introduction


This post aims to provide an overview of ensemble (also called committee-based learning, or learning multiple classifier systems) techniques used in machine learning.

Any ensemble method can be regarded as trying a specific way to decide the weights for combining the individual learners, and different ensemble methods can be regarded as different implementations of weighted averaging.

Benefit of ensemble:

  • Reduce the risk of choosing a wrong hypothesis from hypotheses with same accuracy (problem of high variance)

  • Reduce the risk of choosing a wrong local optima of single learning algorithm (problem of high computational variance)

  • Expand the hypothesis space of representable functions (problem of high bias)
